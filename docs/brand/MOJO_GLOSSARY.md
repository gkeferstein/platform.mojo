# MOJO Glossar

> **Einheitliche Terminologie für das MOJO-Ökosystem** – Konsistente Sprache in allen Apps und Kommunikationen

**Version:** 1.0.0  
**Letzte Aktualisierung:** 01. Januar 2026

---

## Übersicht

Dieses Glossar definiert die verbindliche Terminologie für alle MOJO-Anwendungen. Die konsequente Verwendung dieser Begriffe stärkt die Markenidentität und sorgt für ein einheitliches Nutzererlebnis.

---

## MOJO-Kernbegriffe

Diese Begriffe sind zentral für die MOJO-Identität und sollten in der gesamten Kommunikation verwendet werden.

### LEBENSENERGIE

| Aspekt | Definition |
|--------|------------|
| **Begriff** | LEBENSENERGIE |
| **Deutsch** | Lebensenergie |
| **Definition** | Die fundamentale Kraft, die Menschen befähigt, ihr volles Potenzial zu entfalten. Das Kernkonzept des MOJO-Ökosystems. |
| **Kontext** | Vision, Marketing, User Journey |
| **Beispiel** | "Entdecke deine LEBENSENERGIE und beginne deine Transformation." |

### AHA-Moment

| Aspekt | Definition |
|--------|------------|
| **Begriff** | AHA-Moment |
| **Deutsch** | AHA-Moment |
| **Definition** | Der erste Berührungspunkt mit dem LEBENSENERGIE-Konzept. Der Moment der Erkenntnis, dass Veränderung möglich ist. |
| **Kontext** | Phase 1 der User Journey, Onboarding |
| **Beispiel** | "Dein erster AHA-Moment wartet auf dich." |

### Transformation

| Aspekt | Definition |
|--------|------------|
| **Begriff** | Transformation |
| **Deutsch** | Transformation |
| **Definition** | Die Reise vom erschöpften, ineffektiven Angestellten zum freien, wirksamen, selbstbestimmten Profi. |
| **Kontext** | User Journey, Marketing |
| **Beispiel** | "Deine Transformation nimmt Fahrt auf!" |

### Freischalten

| Aspekt | Definition |
|--------|------------|
| **Begriff** | Freischalten |
| **Deutsch** | Freischalten |
| **Definition** | Progressive Aktivierung von Features und Capabilities im Ökosystem. |
| **Vermeiden** | "Kaufen", "Bezahlen", "Upgraden" |
| **Kontext** | Entitlement-System, Gamification |
| **Beispiel** | "Schalte erweiterte Analytics frei." |

### Level

| Aspekt | Definition |
|--------|------------|
| **Begriff** | Level |
| **Deutsch** | Level |
| **Definition** | Fortschrittsstufe in der User Journey. Jedes Level bringt neue Fähigkeiten und Möglichkeiten. |
| **Kontext** | Gamification, User Journey |
| **Beispiel** | "Du hast Level 2 erreicht! Zeit für den nächsten Schritt." |

### Gesundheitsprofi

| Aspekt | Definition |
|--------|------------|
| **Begriff** | Gesundheitsprofi |
| **Deutsch** | Gesundheitsprofi |
| **Definition** | Die Zielgruppe des MOJO-Ökosystems – Fachkräfte im Gesundheitswesen auf dem Weg zur Selbstständigkeit. |
| **Vermeiden** | "User", "Kunde", "Nutzer" (in Marketingtexten) |
| **Kontext** | Marketing, Kommunikation |
| **Beispiel** | "Als Gesundheitsprofi verdienst du die besten Tools." |

### Ökosystem

| Aspekt | Definition |
|--------|------------|
| **Begriff** | MOJO-Ökosystem |
| **Deutsch** | MOJO-Ökosystem |
| **Definition** | Die Gesamtheit aller MOJO-Apps und -Services, die zusammenwirken. |
| **Vermeiden** | "Plattform", "System" (allein) |
| **Kontext** | Übergreifende Kommunikation |
| **Beispiel** | "Willkommen im MOJO-Ökosystem." |

## Standard-Terminologie

Diese Begriffe sind für die einheitliche UX in allen Apps verbindlich.

### Account & User

| Begriff | Deutsch | Definition | Vermeiden |
|---------|---------|------------|-----------|
| Account | Konto | Das persönliche Benutzerkonto | Profil, User |
| Tenant | Mandant | Arbeitskontext – persönlich oder Organisation | Workspace, Space |
| Organization | Organisation | Gemeinsamer Arbeitsbereich für Teams | Team, Company, Group |
| Member | Mitglied | Ein User, der zu einer Organisation gehört | User, Participant |

### Rollen

| Begriff | Deutsch | Definition |
|---------|---------|------------|
| Owner | Inhaber | Ersteller und Besitzer einer Organisation |
| Admin | Administrator | User mit Verwaltungsrechten |
| Billing Admin | Rechnungs-Administrator | Verantwortlich für Abrechnung |

### Subscription & Billing

| Begriff | Deutsch | Definition | Vermeiden |
|---------|---------|------------|-----------|
| Subscription | Abonnement | Wiederkehrende Zahlung für Zugang | Plan, Package |
| Plan | Tarif | Spezifische Stufe mit definierten Features | – |
| Trial | Testphase | Kostenlose Probephase | Free trial period |

### Navigation

| Begriff | Deutsch | Definition | Vermeiden |
|---------|---------|------------|-----------|
| Dashboard | Dashboard | Hauptübersicht mit Kennzahlen | Home, Overview |
| Settings | Einstellungen | Konfigurationsseiten | Preferences, Options |

### Aktionen

| Begriff | Deutsch | Definition | Vermeiden |
|---------|---------|------------|-----------|
| Invite | Einladen | Einladung zur Organisation senden | Add user |
| Upgrade | Upgraden | Zu höherer Stufe wechseln | Buy, Purchase |
| Downgrade | Downgraden | Zu niedrigerer Stufe wechseln | – |

### Status

| Begriff | Deutsch | Definition |
|---------|---------|------------|
| Active | Aktiv | Gültig und in Verwendung |
| Pending | Ausstehend | Wartet auf Aktion |
| Expired | Abgelaufen | Nicht mehr gültig |
| Locked | Gesperrt | Feature noch nicht freigeschaltet |

## Positiv vs. Negativ Formulierungen

### Entitlements

| ❌ Nicht verwenden | ✅ Stattdessen |
|-------------------|----------------|
| "Du kannst nicht..." | "Schalte XY frei, um..." |
| "Nicht verfügbar in deinem Plan" | "Verfügbar ab Level/Tarif XY" |
| "Zugriff verweigert" | "Dieses Feature wartet auf dich" |
| "Kaufe Premium" | "Schalte Premium frei" |

### Fehler

| ❌ Nicht verwenden | ✅ Stattdessen |
|-------------------|----------------|
| "Fehler aufgetreten" | "Etwas hat nicht geklappt" |
| "Ungültige Eingabe" | "Bitte überprüfe deine Eingabe" |
| "Aktion fehlgeschlagen" | "Das hat leider nicht funktioniert" |

### Erfolg

| ❌ Nicht verwenden | ✅ Stattdessen |
|-------------------|----------------|
| "Erfolgreich gespeichert" | "Perfekt, gespeichert!" |
| "Aktion abgeschlossen" | "Erledigt!" |
| "Änderungen übernommen" | "Deine Änderungen sind sicher." |

## Verwendung in Code

Das `@mojo/ux` Package in `design.mojo` enthält diese Terminologie als TypeScript-Definitionen:

```typescript
import { getTerm, getGermanTerm, mojoTerminology } from '@mojo/ux'

// MOJO-spezifische Begriffe
const lebensenergie = mojoTerminology.lebensenergie.term // "LEBENSENERGIE"

// Standard-Terminologie
const accountDe = getGermanTerm('account') // "Konto"
```
